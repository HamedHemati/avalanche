class avalanche.training.storage_policy.ExperienceBalancedStoragePolicy(ext_mem: Dict, mem_size: int, adaptive_size: bool = True, num_experiences=- 1)[source]

Bases: avalanche.training.storage_policy.StoragePolicy

A policy to store exemplars in a replay memory.

Parameters

  • ext_mem – The replay memory dictionary to store samples.

  • mem_size – max number of total input samples in the replay memory.

Stores samples for replay, equally divided over experiences. Because it is conditioned on the experience, it should be called in the ‘after_training_exp’ phase.

The number of experiences can be fixed up front or adaptive, based on the ‘adaptive_size’ attribute. When adaptive, the memory is equally divided over all the unique observed experiences so far.

Parameters

  • ext_mem – The replay memory dictionary to store samples.

  • mem_size – max number of total input samples in the replay memory.

  • adaptive_size – True if mem_size is divided equally over all observed experiences (keys in replay_mem).

  • num_experiences – If adaptive size is False, the fixed number of experiences to divide capacity over.

Stores samples for replay, equally divided over classes. It should be called in the ‘after_training_exp’ phase (see ExperienceBalancedStoragePolicy). The number of classes can be fixed up front or adaptive, based on the ‘adaptive_size’ attribute. When adaptive, the memory is equally divided over all the unique observed classes so far. :param ext_mem: The replay memory dictionary to store samples in. :param mem_size: The max capacity of the replay memory. :param adaptive_size: True if mem_size is divided equally over all
